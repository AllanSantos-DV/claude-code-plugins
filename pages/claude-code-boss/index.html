O que entrega

Brain KB

Base de conhecimento local

Busca semântica sobre SQLite e embeddings. Indexa o que importa e recupera contexto antes de cada ação, com rerank por relevância, recência e frequência.

Backend opcional

Servidor MCP Memory externo

Troque o SQLite local por um servidor de memória seu ou da equipe. A ativação é por usuário, sobrevive ao auto-update e não afeta quem não usa.

Identidade de projeto

Recall que casa entre máquinas

O escopo vem de um id estável: CCB_PROJECT_ID, depois o arquivo .claude-boss-project, depois o nome da pasta. A memória segue o projeto, não o caminho.

Aprendizado no loop

Lições sem reler o transcript

A tool capture_lesson destila lições no próprio turno, mescla duplicatas e promove padrões recorrentes a skills.

Execução curada

Sinal acima de ruído

Redireciona comandos ruidosos e injeta contexto de forma advisory, com backpressure para não inflar a janela.

Dashboard

Inspeção sob demanda

Um painel em localhost para ver a KB, os hooks e os logs. Abre quando você chama, não no boot.

Como funciona

Você escreve um prompt. O hook UserPromptSubmit recupera do Brain e injeta um bloco de contexto do projeto.

O agente age com a memória do projeto à mão, escopada pelo id estável do projeto.

Ao terminar, os hooks de Stop destilam lições e curam comandos ruidosos. Nada disso relê o transcript nem gasta a janela.
