O que entrega

Brain KB

Base de conhecimento local

Busca semântica sobre SQLite e embeddings. Indexa o que importa e recupera contexto antes de cada ação, com rerank por relevância, recência e frequência. É o backend padrão e segue inalterado na v2.0.0.

Backend opcional

Servidor MCP Memory externo

Troque o SQLite local por um servidor de memória seu ou da equipe. A ativação é por usuário, sobrevive ao auto-update e não afeta quem não usa. Na v2.0.0 ele fala o contrato compose/ingest — exige memory-server ≥ 2.18.

Identidade de projeto

Recall que casa entre máquinas

O escopo vem de um id estável: CCB_PROJECT_ID, depois o arquivo .claude-boss-project, depois o nome da pasta. A memória segue o projeto, não o caminho — e, na v2.0.0, não vaza para outro projeto.

Aprendizado no loop

Lições sem reler o transcript

A tool capture_lesson destila lições no próprio turno, mescla duplicatas e promove padrões recorrentes a skills.

Execução curada

Sinal acima de ruído

Redireciona comandos ruidosos e injeta contexto de forma advisory, com backpressure para não inflar a janela.

Dashboard

Inspeção sob demanda

Um painel em localhost para ver a KB, os hooks e os logs. Abre quando você chama, não no boot.

Novo na v2.0.0 · recall em dois níveis

A v2.0.0 muda como o boss lembra. No backend externo, o recall vem em dois níveis: os fatos relevantes entram com o trecho que casou — a resposta fica fundamentada; as capacidades entram como ponteiros leves, que o agente abre só quando precisa.

Breaking change No backend externo mcp-memory, o recall agora usa compose_recall e exige memory-server ≥ 2.18 — o search_memory raso sai do caminho. O backend local (SQLite) fica intacto: quem usa local não muda nada.

Ingestão cooperativa

No Stop, o transcript acumulado da sessão vai pro ingest_conversation. O servidor descobre o template, destila, tipa, escopa e deduplica por event-id — o cliente só envia, reenviar o acumulado é seguro.

Sem vazamento entre projetos

O brain-backend.init() virou project-aware: em outro projeto refaz o handshake e re-escopa, em vez de reusar o primeiro. Chamada sem projeto nunca sobrescreve o escopo atual.

Falha visível (fail-loud)

O recall-health marca cada recall como ok ou degradado; o brain-health avisa no SessionStart quando o recall vem vazio de forma recorrente — daemon < 2.18 ou fora do ar.

Orçamento de injeção + knobs

Em kb.retrieval.compose: includeHomeSpine, maxInjectChars (teto de texto), timeoutMs (compose lento degrada) e um overlay de metadados genérico.

Tudo atrás de gate e testado — 65 suítes de hooks + 380 unitários no verde, com smoke real contra o daemon 2.18.

Como funciona

Você escreve um prompt. O hook UserPromptSubmit chama o Brain e injeta o bloco [BRAIN]: no backend externo, os fatos vêm com o trecho casado e as capacidades como ponteiros.

O agente age com a memória do projeto à mão, escopada por um id estável — e, na v2.0.0, sem puxar contexto de outro projeto.

No Stop, o boss destila o aprendizado e cura comandos ruidosos. No backend externo, manda o transcript acumulado pro servidor destilar. Nada relê o transcript à toa nem infla a janela.
